Ever wanted to know exactly what your Salad app is doing behind the scenes?

This is a lightweight, local web dashboard that parses your Salad logs to give you real-time insights into your machine's compute and bandwidth sharing performance. Instead of digging through raw text files, this tool visualizes your wallet balance, container earnings, and bandwidth usage in clean, interactive charts.

# Features
Real-Time Charts: Tracks your Wallet Balance, Container Earnings, and Bandwidth Sharing (MB/s) over the last 24 hours using Chart.js.

Container History: A sliding sidebar that shows you exactly which workloads are running, when they started, and their projected 24-hour payout rate.

Smart Error Notifications: Non-intrusive toast pop-ups automatically alert you to workload failures or node compatibility issues so you can step in if needed.

System Info: Automatically detects and displays your current CPU, GPU, RAM, and OS straight from the logs.

Modern UI: A clean, responsive design built with Bootstrap 5, complete with a built-in Dark/Light mode toggle.

# Prerequisites
You'll need Python installed on your machine to run the backend server. You will also need to install a few standard Python libraries to handle the web server and data sanitization:

``` bash
pip install flask flask-cors bleach
```
 Getting Started
Clone this repository or download server.py and index.html to the same folder on your machine.

Open your terminal or command prompt in that directory.

Start the server by running the following command (replace "YourMachineName" with whatever you want to call your rig):

``` bash
python server.py -machine_name "YourMachineName"
```
(Note: The server defaults to port 8000, but you can specify a custom port by adding -port 8080).

Open your web browser and navigate to http://localhost:8000.

# How it Works
The Backend (server.py): Acts as a local REST API. It actively reads the raw logs generated by Salad in C:\ProgramData\Salad\logs, uses regex to extract the important data points (like BidirThroughput, Predicted Earnings, and Workload Failures), and serves it up as clean JSON. It uses a caching system so it only processes new log lines, keeping it incredibly fast.

The Frontend (index.html): A single-page application that ping the local API every 60 seconds. It intelligently groups your data by Container ID and dynamically renders your charts, history sidebar, and toast notifications without needing to refresh the page.

A Note on Privacy
This tool runs 100% locally on your machine. It simply reads the logs already present on your hard drive and serves them to your local browser. No data is sent to external servers.
